### [Dreamer: The Personal Agent OS](https://www.latent.space/p/dreamer)

/dev/agents came out of stealth as Dreamer — a consumer-first platform for building, discovering, and running AI agents, led by ex-Stripe CTO David Singleton and ex-Android co-founder Hugo Barra. The core "Sidekick" is an agent that builds other agents in natural language, on top of a full proprietary stack: SDK, logging, database, serverless functions, and an open tool marketplace where builders get paid per usage.

> The full-stack commitment here is the thing to notice. Enterprise teams chasing the same "agent OS" vision are typically stitching together six different vendors. Dreamer does it in one, and they're paying tool builders on usage — which is a real ecosystem flywheel if it works.

### [AgentArmor: 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

Open-source framework that wraps any agentic architecture with 8 security layers: prompt injection detection, encrypted storage, context separation, action risk scoring (READ=1 → DELETE=7 → EXECUTE=8 → ADMIN=10), network egress control, PII redaction via Microsoft Presidio, HMAC inter-agent authentication, and JIT identity. Tested against the OWASP ASI spec from December 2025.

> Security is the obvious gap in most production agent builds right now — most teams ship with no guardrails beyond "we trust the model." Having action risk scoring and human approval gates as first-class primitives is exactly what enterprise deployments need, and it's good that the open-source community is moving on this before the incidents force it.

### [Cursor's New Coding Model Was Built on Moonshot AI's Kimi](https://techcrunch.com/2026/03/22/cursor-admits-its-new-coding-model-was-built-on-top-of-moonshot-ais-kimi/)

Cursor disclosed its new coding model is fine-tuned from Kimi, a Chinese model from Moonshot AI — after users noticed anomalous behavior and pushed for answers. The admission came after community pressure.

> The coding assistant market is increasingly opaque about what's actually running under the hood. From an enterprise governance standpoint this matters: companies with data residency rules, geopolitical restrictions, or IP sensitivity need to know exactly what model is processing their code. "Trust us" isn't a compliance posture.

### [Bot Traffic Will Exceed Human Web Traffic by 2027 — Cloudflare CEO](https://techcrunch.com/2026/03/19/online-bot-traffic-will-exceed-human-traffic-by-2027-cloudflare-ceo-says/)

> This is the infrastructure story most AI architects aren't accounting for yet. When your agentic workflows are one of thousands hitting the same APIs simultaneously, rate limiting, caching, and egress cost models look fundamentally different. It changes how you design agent tool calls at scale.

### [Using Git with Coding Agents](https://simonwillison.net/guides/agentic-engineering-patterns/using-git-with-coding-agents/)

Simon Willison published a practical guide on how coding agents like Claude Code and Codex use Git — covering branching, bisect, history rewriting, merge conflict resolution, and library extraction. The core point: agents are fluent enough with Git that you can offload the most painful institutional knowledge (rebase, reflog recovery, cherry-pick conflicts) to them entirely.

> This is the kind of workflow multiplier that matters in enterprise codebases where Git history is actually load-bearing for compliance and audit trails. Worth reading as a template for how to brief agents on existing repos.

### [Experimenting with Starlette 1.0 with Claude Skills](https://simonwillison.net/2026/Mar/22/starlette/)

Starlette 1.0 just shipped, breaking compatibility with prior versions in its lifespan API. Willison used Claude's new skill system to build a comprehensive skill document from the Starlette repo, then used that skill to generate a working task management app — with Claude testing the code itself. A new "Copy to your skills" button in the Claude UI streamlines this loop considerably.

> The broader signal: LLMs with up-to-date skills for specific frameworks close a major reliability gap. If you're building .NET + AI apps and tired of Copilot generating deprecated SDK calls, this skill-as-context-injection pattern is worth adopting systematically.
