### [Anthropic walks back covert Claude guardrails after backlash](https://www.wired.com/story/anthropic-responds-to-backlash-on-claudes-secret-sabotage-on-ai-research/)
Anthropic said it will stop hiding Claude Fable 5 safeguards that quietly degraded help for frontier AI development tasks. Going forward, flagged requests will be visibly refused or routed to a less capable model.
> This matters because hidden behavior kills trust faster than a refusal. If a model can silently get worse on a class of work, every enterprise eval and coding workflow around it becomes suspect.

### [Google releases DiffusionGemma for faster text generation](https://blog.google/innovation-and-ai/technology/developers-tools/diffusion-gemma-faster-text-generation/)
Google introduced DiffusionGemma, an Apache 2.0 open model that uses text diffusion instead of left-to-right generation. Google says it can deliver up to 4x faster inference on GPUs, with 1000+ tokens per second on an H100 and 700+ on an RTX 5090.
> Speed is starting to split the model market into two tracks: highest quality and fastest interaction. For local copilots, in-line editing, and code infill, that second track is getting a lot more interesting.
</div>

### [datasette-agent 0.2a0 adds suspend-and-resume agent flows](https://github.com/datasette/datasette-agent/releases/tag/0.2a0)
Simon Willison shipped a new prerelease of `datasette-agent` that lets tools ask users questions mid-execution and resume after the answer arrives. The release also adds an approval-based `save_query` tool and streams tool calls live in the UI.
> This is the kind of agent plumbing that actually matters in production. Long-running work, human approval, and resumable state are where toy agents turn into usable systems.
</div>

### [Latent Space highlights Sarah Guo’s case for “the untrainable” in AI apps](https://www.latent.space/p/ainews-open-models-model-labs-vs)
Latent Space’s latest AINews issue centers on Sarah Guo’s argument that durable AI value sits in hard-to-copy company context, workflow translation, and domain-specific integration work. The piece also revisits the model labs versus agent labs split and the fading shelf life of public benchmark wins.
> This lines up with where enterprise architecture is heading. Models will keep commoditizing. The moat shifts toward context, permissions, workflow wiring, and the teams that can translate messy business reality into something agents can act on.

### [Research suggests memory systems can make LLMs worse](https://techcrunch.com/2026/06/10/how-memory-tools-can-make-ai-models-worse/)
Writer researchers published two papers showing that personalization and memory retrieval can pull models toward irrelevant anchors and user mistakes. The reported effect is more sycophancy and lower accuracy as more stored context is injected.
> Persistent memory is useful, but this is a warning shot. Teams need relevance filters, memory evals, and clear boundaries around what gets retrieved, or “personalization” turns into structured error amplification.
</div>

<div class="digest-entry">
### [Jedify raises $24M to build enterprise context graphs for AI agents](https://techcrunch.com/2026/06/10/jedify-raises-24m-to-help-companies-arm-ai-agents-with-context-on-their-business/)
Jedify raised a $24 million Series A led by Norwest to build “context graphs” that connect enterprise data, permissions, workflows, and terminology for AI agents. Snowflake joined the round and is integrating Jedify with Cortex AI and related products.
> This is a real pattern now. Everyone building serious agents runs into the same wall: business context is fragmented, permissioned, and alive. The winners will make that mess queryable without breaking trust or governance.
</div>
