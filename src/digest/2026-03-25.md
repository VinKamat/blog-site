### [Anthropic Hands Claude Code More Control with Auto Mode](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/)

Anthropic shipped auto mode for Claude Code, a new permissions layer that replaces the blunt `--dangerously-skip-permissions` flag. A separate Claude Sonnet 4.6 classifier runs before each action and decides whether it falls within the intended task scope — blocking anything that escalates beyond stated intent, targets unrecognized infrastructure, or looks like it was driven by hostile content in a file or webpage.

> This is the right direction. Agentic coding tools need something between "ask every time" and "do whatever." The classifier-as-guardrail pattern is interesting — using a secondary model to audit the primary agent's actions is a pattern we'll see a lot more of in enterprise deployments. Simon Willison's skepticism about non-deterministic prompt injection defenses is valid: you still want a deterministic sandbox underneath this for anything sensitive.

### [LiteLLM 1.82.8 Supply Chain Attack — Credential Stealer Hidden in PyPI Package](https://simonwillison.net/2026/Mar/24/malicious-litellm/)

LiteLLM versions 1.82.7 and 1.82.8 were compromised with a credential stealer that targeted SSH keys, AWS configs, Azure credentials, Docker tokens, shell histories, and more. The v1.82.8 variant used a `.pth` file, meaning the malware ran at Python startup — import wasn't even required. The attack traced back to a compromised Trivy (security scanner) GitHub Action used in LiteLLM's own CI.

> Any enterprise using LiteLLM in an AI pipeline should audit whether these versions touched their systems. The attack vector — compromising the security tool used to protect the package — is particularly nasty. This lands right as coding agents are auto-installing dependencies with minimal review, which makes the timing worse. Dependency cooldowns (now supported by pnpm, Yarn, Bun, Deno, uv, pip, and npm) are worth enabling on AI project pipelines.

### [Figma's MCP Server Puts Design Canvases Directly in Agents' Hands](https://www.latent.space/p/ainews-apples-war-on-slop)

Figma launched an MCP server in open beta, enabling AI agents to read and edit design canvases directly. Cursor extended this to generate components using a team's design system. GitHub Copilot and other MCP-compatible clients can reach it through the same interface. Latent Space called it one of the clearest examples of tool-calling becoming product-native rather than chat-wrapper-native.

> This is what the MCP ecosystem was built for. When design tools expose a standard interface, agents stop being chat wrappers and start being real participants in the software development workflow. The design-to-implementation loop — Figma → agent → code — shortens significantly. Worth watching for enterprise design system governance implications: agents pulling from shared design tokens is powerful but needs guardrails.

### [Hermes Agent v0.4.0 — 300 PRs in a Week, Now OpenAI API Compatible](https://www.latent.space/p/ainews-apples-war-on-slop)

Nous Research shipped a massive Hermes Agent update with an OpenAI-compatible Responses API backend, background self-improvement loops, broader messaging integrations, and improved context compression. The most interesting feature: a post-response review agent that decides what to retain as reusable memory or skills. The OpenAI API compatibility means it's usable from Open WebUI, LobeChat, or any compatible client.

> The memory-as-skill-retention pattern is worth studying. Agents that accumulate structured knowledge from their own runs, rather than relying on static system prompts or vector retrieval, are closer to how we'd want production agentic systems to evolve. The OpenAI API compatibility layer is smart — it lowers adoption friction to near zero for teams already on that stack.

### [AgentArmor — Open-Source 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

An open-source framework that wraps agentic architectures with 8 independent security layers: prompt injection detection, vector DB encryption, instruction-data separation, action risk scoring (READ=1 to ADMIN=10), network egress control, PII redaction via Microsoft Presidio, inter-agent HMAC auth, and agent-native JIT identity. Tested against all 10 OWASP Agentic Security Integrity risks from the December 2025 spec.

> The action risk scoring layer (L4) is the part most production teams skip. Agents reading emails and calling APIs with essentially no risk model is the norm right now, and it's going to bite people. This framework is early-stage but the OWASP ASI alignment gives it a useful taxonomy. Worth evaluating for any team shipping agentic systems that touch sensitive enterprise data.

### [OpenAI's Sora App Shuts Down](https://techcrunch.com/2026/03/24/openais-sora-was-the-creepiest-app-on-your-phone-now-its-shutting-down/)

OpenAI is shutting down the standalone Sora mobile app. The underlying Sora 2 video and audio generation model remains, but the AI-only social feed never found sustained traction. Latent Space noted it as the first casualty of OpenAI's "Side Quest" product pruning.

> The lesson here isn't that generative video failed — it's that bolting a social feed onto a generation model doesn't automatically create a product. Sora 2's capabilities are real; the distribution strategy wasn't. Expect the model to resurface embedded in other OpenAI products rather than disappear entirely.
