### [LiteLLM Malware Attack: 47,000 Downloads in 46 Minutes](https://futuresearch.ai/blog/litellm-hack-were-you-one-of-the-47000/)
A malicious version of LiteLLM (1.82.7 and 1.82.8) was live on PyPI for 46 minutes before being pulled — long enough for 46,996 downloads. Analysis via BigQuery's PyPI dataset found 2,337 dependent packages, 88% of which didn't pin versions.

> Supply chain attacks on LLM infrastructure aren't theoretical anymore. LiteLLM sits in a lot of enterprise AI stacks as a routing layer, and this was a targeted, well-timed hit. If your organization runs any AI services via pip, this is a wake-up call to audit your dependency pinning policy — and your CI security posture. The fact that an AI assistant (Claude) helped identify and triage the vulnerability in real time is its own interesting footnote.

### [Thoughts on Slowing the Fuck Down (Agentic Engineering)](https://news.ycombinator.com/item?id=47517539)
Mario Zechner, creator of the Pi agent framework, argues that coding agents remove human bottlenecks in ways that let bugs and complexity compound far faster than any individual could accumulate them. The post calls for deliberate limits on daily agent output and keeping architecture decisions hand-written.

> This hits something real. Agents can ship 20,000 lines of code before you've finished your coffee, but the cognitive debt accumulates at the same rate. Enterprise codebases are already hard to reason about — let an agent run unchecked for a week and you may not recognize it. The discipline needed now is less about coding speed and more about comprehension speed: how fast can you actually absorb and own what's been generated? That's the real constraint architects need to design around.

### [Stripe Launches Projects.dev — Agent-Native Service Provisioning](https://www.latent.space/p/ainews-everything-is-cli)
Stripe introduced Projects.dev, a CLI that lets agents provision third-party services (PostHog, Vercel, etc.) in a single command — e.g. `stripe projects add posthog/analytics`. Inspired by Andrej Karpathy's MenuGen and part of a broader trend toward CLI-first agent infrastructure.

> The interesting thing here isn't Stripe acting as a service marketplace — it's that agent-native infra is crystallizing around CLIs as the universal interface. Agents work best with structured, composable commands, and CLIs are exactly that. For enterprise architects thinking about agentic workflows that touch external services, this is a preview of how provisioning and integration will look in 2-3 years. The billing and auth implications are significant.

### [AgentArmor — 8-Layer Open-Source Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)
AgentArmor wraps agentic architectures with eight independent security layers covering prompt injection, vector DB encryption, context separation, action risk scoring, egress control, PII redaction, inter-agent auth, and identity/JIT permissions. Validated against the full OWASP ASI (Agentic Security Integrity) December 2025 spec.

> Most production agent deployments right now rely on "trust the LLM" as their entire security model — which is not a security model. AgentArmor is one of the first structured approaches that maps to an actual spec (OWASP ASI). The L4 action risk scoring layer alone — READ=1 through ADMIN=10 with chain depth limits — is the kind of control plane that enterprise security teams will eventually require before signing off on any agentic deployment. Worth watching this project.

### [Google TurboQuant: 6x AI Memory Compression](https://techcrunch.com/2026/03/25/google-turboquant-ai-memory-compression-silicon-valley-pied-piper/)
Google's TurboQuant algorithm claims up to 6x compression of AI "working memory" (KV cache). Still a lab experiment, but the internet has already made the Pied Piper comparisons. Related to the broader quantization story covered in Sam Rose's interactive essay on model weight quantization.

> KV cache is one of the real cost drivers at inference time, especially for long-context enterprise workloads. 6x compression on that layer would meaningfully change the economics of running large models in production — fewer GPUs, lower latency, smaller per-token cost. Don't read too much into lab numbers, but the direction is right. Pair this with the quantization deep dive from Sam Rose and you get a clear picture of where AI infrastructure efficiency is heading.
