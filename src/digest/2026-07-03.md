### [llm-coding-agent 0.1a0](https://simonwillison.net/2026/Jul/2/llm-coding-agent/)
Simon Willison published an alpha Python coding agent built on his evolving `llm` framework and shipped it to PyPI. The package exposes a small toolset for file edits, command execution, file listing, search, and file reads, along with a simple Python API.
> This is a clean signal that lightweight agent frameworks are getting real enough to package and ship. The interesting part is the tool contract: explicit file edit, command execution, and bounded permissions. That is the layer enterprises will standardize first.

### [Using DSPy to evaluate and improve Datasette Agent's SQL system prompts](https://simonwillison.net/2026/Jul/2/dspy-datasette-agent-prompts/)
Willison also shared a DSPy-based evaluation project for Datasette Agent's read-only SQL workflow. The work uses real tools and a gold-standard dataset to test prompt behavior, and it surfaced concrete prompt issues like weak schema detail causing column-name guessing and retry loops.
> This is the kind of work that matters in production. Prompt quality moves when you put it under a harness, use real tools, and inspect failure traces. Every enterprise agent team needs this loop for SQL, retrieval, and workflow prompts.

### [AIEWF Daily Dispatch: The great loops debate and the state of AI engineering](https://www.latent.space/p/aiewf-daily-dispatch-locomotives)
Latent.Space recapped the closing day of the AI Engineer World's Fair, where speakers argued over autonomous coding loops, software factories, and the limits of current engineering discipline. The piece also cites Amplify survey data showing 95% of respondents now use agents, 89% of agent-using teams let agents write data, and many teams still rely on human approvals as their main safeguard.
> That survey lines up with what serious teams are seeing: action-taking agents are here before governance is mature. The gap is the control plane. Permissions, review, tracing, and rollback will decide who gets past demo stage.

### [Microsoft launches its own AI deployment company with $2.5 billion commitment](https://techcrunch.com/2026/07/02/microsoft-launches-its-own-ai-deployment-company-with-2-5-billion-commitment/)
Microsoft announced Microsoft Frontier Company, a new operating business focused on enterprise AI deployments using Microsoft's existing stack. Microsoft says it will back the effort with $2.5 billion and 6,000 industry and engineering experts, with early named partners including London Stock Exchange Group, Unilever, Land O'Lakes, and Accenture.
> Microsoft is moving from platform vendor to delivery engine. That matters for Azure shops because architecture choices, reference patterns, and operating models may now get pushed through services engagements faster than through product docs.

### [Cloudflare's new policy pushes AI companies to pay for publishers' content](https://techcrunch.com/2026/07/01/cloudflares-new-policy-pushes-ai-companies-to-pay-for-publishers-content/)
Cloudflare said that starting September 15, 2026, its default settings will block mixed-use crawlers from ad-supported pages unless site owners opt in. The policy is meant to force clearer separation between search crawling and AI training or agent use, and it expands Cloudflare's publisher monetization model from Pay Per Crawl toward Pay Per Use.
> This could reshape the retrieval stack around licensed and clearly labeled access paths. Teams building agent products on public web data should expect more negotiation, more metering, and fewer assumptions about free crawl access.
</div>
