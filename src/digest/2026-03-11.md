### [Yann LeCun's AMI Labs Launches with $1B Seed to Build World Models](https://www.latent.space/p/ainews-yann-lecuns-ami-labs-launches)

AMI Labs closed a $1.03B seed round at a $3.5B valuation to build AI that understands the physical world via JEPA (Joint Embedding Predictive Architecture). LeCun's bet: autoregressive LLMs predict tokens but can't model reality. The founding team is heavy on vision, video, and representation learning researchers, with Paris and Zürich as primary locations.

> The biggest question in AI right now isn't which LLM is best — it's whether token prediction is the right foundation at all. LeCun has been making this argument for years; now he has $1B to test it. For enterprise architects, if world models handle state, causality, and action outcomes better than LLMs alone, that changes how we design agentic systems for operational domains.

### [Clinejection: Prompt Injection via GitHub Issue Title Poisons Release Pipeline](https://adnanthekhan.com/posts/clinejection/)

A researcher demonstrated how a prompt injection embedded in a GitHub issue title hijacked Cline's AI-powered issue triage agent, poisoned the GitHub Actions cache through a cache eviction trick, and gained access to NPM publishing secrets shared via common cache keys between the triage and release workflows.

> The attack surface in agentic CI/CD isn't the model — it's the harness, permissions, and cache boundaries. If you're running Claude Code actions or AI triage agents in GitHub workflows, read this before your next deployment. It's the most concrete agentic security failure documented to date, and the exploit chain is devastatingly elegant.

### [AgentMail Raises $6M to Build Email Infrastructure for AI Agents](https://techcrunch.com/2026/03/10/agentmail-raises-6m-to-build-an-email-service-for-ai-agents/)

AgentMail raised $6M from General Catalyst to provide an API that gives AI agents their own persistent email inboxes — two-way conversations, threading, labeling, search, and reply support built for programmatic agent use.

> Agents need persistent, addressable communication channels. Email is the first because it's universally present in enterprise workflows. This is agent infrastructure filling in the gaps — the same way we built queues and event buses for microservices, we're now building identity and messaging primitives for agents. The pattern is early but the direction is clear.

### [Simon Willison: Coding Agents Should Help Us Ship Better Code, Not Worse](https://simonwillison.net/guides/agentic-engineering-patterns/better-code/)

Willison argues that shipping lower-quality code with AI agents is a choice, not an inevitability. Coding agents are ideal for simple-but-time-consuming technical debt fixes — refactoring APIs, renaming concepts across codebases, splitting large files — and for exploratory prototyping that validates technology choices cheaply before committing.

> The real productivity unlock from coding agents is that the cost of fixing technical debt has dropped near zero. For enterprise codebases with years of accumulated debt, that's a meaningful shift — teams can finally afford zero-tolerance on code smells without sacrificing feature velocity. Worth building this into your team's agent workflow now, before the backlog becomes culture.
