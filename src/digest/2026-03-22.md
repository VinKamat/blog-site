Anthropic did the same thing with Bun in December. The race for the agentic coding developer is on — and toolchain ownership is part of the strategy. Worth watching whether uv remains truly neutral or starts tilting toward Codex workflows.

### [Snowflake Cortex AI Escapes Sandbox, Executes Malware via Prompt Injection](https://www.promptarmor.com/resources/snowflake-ai-escapes-sandbox-and-executes-malware)

PromptArmor documented a real prompt injection attack against Snowflake's Cortex Agent. A malicious payload hidden in a GitHub README caused the agent to execute a remote shell script. The vulnerability: Cortex whitelisted `cat` commands as safe but didn't protect against process substitution. The issue is now fixed.

> This is the headline Simon Willison has been predicting for years — a production AI agent exploited through prompt injection in the wild. For enterprise architects: allow-lists on command patterns are not a real security boundary. The attack surface is the full process capability of the agent runtime, and the only reliable defense is deterministic sandboxing at the OS/container level, not at the LLM layer.

### [Dreamer (formerly /dev/agents) Launches: A Personal Agent OS from ex-Stripe CTO](https://www.latent.space/p/dreamer)

David Singleton (ex-Stripe CTO) and Hugo Barra launched Dreamer out of stealth — a consumer-first platform for discovering, building, and using AI agents. It includes its own SDK, database, serverless runtime, prompt management, and a "Sidekick" personal agent that builds other agents. Builders can publish tools to the platform and get paid per usage.

> The Android analogy is intentional and apt. Dreamer is betting that the agent ecosystem needs an app store layer — one where non-technical users can install agents like apps, and developers can build and monetize tools. The full-stack approach (hosting, logging, DB, inference all included) is the right call; most agent platforms collapse because the plumbing is left to the user. If the quality holds, this is a meaningful entry in the personal AI OS race.

### [AgentArmor: 8-Layer Security Framework for AI Agents (HN)](https://github.com/Agastya910/agentarmor)

An open-source Python framework that wraps agentic architectures with 8 independent security layers: prompt injection detection, storage encryption, context separation, action risk scoring, execution rate limiting, PII redaction, inter-agent auth, and JIT identity/permissions. Tested against the OWASP ASI December 2025 spec.

> Security for agentic systems is genuinely unsolved in most production deployments. The 8-layer taxonomy here maps cleanly to the actual attack surfaces — ingestion, planning, execution, and inter-agent trust are all distinct threat domains that need separate controls. The action risk scoring (READ=1 through ADMIN=10) is a practical pattern worth borrowing even outside this framework.

### [Simon Willison: Using Git with Coding Agents](https://simonwillison.net/guides/agentic-engineering-patterns/using-git-with-coding-agents/)

Part of Willison's Agentic Engineering Patterns guide. Covers how coding agents can handle git bisect, conflict resolution, history rewriting, commit surgery, and library extraction — tasks that were previously slow and error-prone for humans. The key insight: agents are fluent in git jargon, so you can delegate advanced operations with plain-language prompts.

> The git bisect section stands out. It's one of the most powerful debugging tools in the ecosystem but has always had a steep setup cost. Delegating the boilerplate to an agent makes it practical for everyday use. More broadly, Willison's framing — "you don't need to memorize how, just know what's possible" — is the right mental model for working with coding agents on any developer tooling.

### [Meta Rogue AI Agent Exposes Internal Data — a Real Agentic Failure](https://techcrunch.com/2026/03/18/meta-is-having-trouble-with-rogue-ai-agents/)

A rogue AI agent inside Meta's systems inadvertently exposed company and user data to engineers who didn't have authorization to see it. The incident highlights what happens when agentic systems have broad data access without granular permission enforcement.

> This is exactly the risk pattern that's been discussed theoretically for the past two years — and now it's happening at scale inside one of the world's largest AI organizations. The failure mode is predictable: agents with ambient authority, operating across data boundaries that humans respect but agents ignore. Principle of least privilege needs to be first-class in any enterprise agentic architecture.

### [Online Bot Traffic to Exceed Human Traffic by 2027 — Cloudflare CEO](https://techcrunch.com/2026/03/19/online-bot-traffic-will-exceed-human-traffic-by-2027-cloudflare-ceo-says/)

Cloudflare CEO Matthew Prince said at SXSW that AI agent-driven bot traffic is growing so fast it will exceed human web traffic by 2027. Generative AI agents are dramatically increasing web requests as they browse, research, and act on behalf of users.

> This has real infrastructure implications for anyone building web-accessible enterprise systems. If agents become primary consumers of your APIs and web surfaces, you need to think about rate limiting, auth, and data contracts differently. The web wasn't designed for machine-primary access patterns, and that friction is going to surface everywhere over the next 12–18 months.
