### [1M context is now generally available for Opus 4.6 and Sonnet 4.6](https://claude.com/blog/1m-context-ga)

Anthropic has made 1M context windows generally available for both Opus 4.6 and Sonnet 4.6, with standard pricing across the full window — no long-context premium. OpenAI charges more above 272K tokens (GPT-5.4) and Gemini above 200K, making this a meaningful pricing differentiator. Media limits are also expanded: 600 images or PDF pages per request.

> The no-premium pricing is the real story here. For enterprise workloads that routinely push large codebases, contracts, or document sets into context, this removes a cost ceiling that was quietly shaping architecture decisions. Long-context is no longer a luxury tier — Anthropic just made it the default.

### [Shopify CEO Uses Agentic Autoresearch to Get 53% Faster Parse+Render in Liquid](https://simonwillison.net/2026/Mar/13/liquid/)

Tobi Lütke used Andrej Karpathy's autoresearch pattern — a coding agent running hundreds of semi-autonomous experiments — to find performance improvements in Shopify's open-source Ruby templating engine Liquid. 120 automated experiments across 93 commits. 53% faster parse+render, 61% fewer allocations. The benchmark suite made "make it faster" into an actionable, verifiable goal.

> This is agentic engineering at its clearest: give the agent a test suite, a benchmark script, and a measurable goal. The autoresearch loop — brainstorm, experiment, commit, evaluate — maps directly onto how you'd want an AI agent operating in a CI-adjacent workflow. The pattern transfers well beyond Ruby; any enterprise codebase with good test coverage is a candidate.

### [Coding After Coders: The End of Computer Programming as We Know It](https://www.nytimes.com/2026/03/12/magazine/ai-coding-programming-jobs-claude-chatgpt.html)

NYT Magazine's Clive Thompson interviewed 70+ software developers at Google, Amazon, Microsoft, and Apple about AI-assisted development. The general tone from those interviewed was optimistic — with a nod to Jevons paradox potentially increasing overall demand for software development. The piece also surfaces a quieter split: engineers who use AI for velocity vs. those who mourn the craft.

> The "craft vs. output" tension will keep resurfacing. In enterprise settings that tension gets resolved quickly — delivery pressures are real, and agents are now fast enough that the choice is less "should we use them" and more "how do we govern them." The more interesting question from the piece is whether demand expands enough to absorb the productivity gains. Historical precedent says yes, but this inflection feels different in pace.

### [AgentArmor – Open-Source 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

Published today on HN, AgentArmor wraps agentic architectures with 8 independent security layers targeting distinct attack surfaces in an agent's data flow: prompt injection, storage encryption, context isolation, action risk scoring, execution guardrails, PII redaction, inter-agent auth, and JIT identity/permissions. Tested against all 10 OWASP ASI (Agentic Security Integrity) risks from the December 2025 spec.

> Most teams building production agents are still trusting the LLM by default. AgentArmor formalizes the security layers that should already exist — especially L4 (action risk scoring) and L5 (human approval gates), which map cleanly to enterprise change-control requirements. The OWASP ASI spec backing is a credibility marker. Worth evaluating as a baseline before your next agentic deployment review.

### [Persistent Memory Is Becoming the Differentiator for AI Agents](https://www.latent.space/p/ainews-context-drought)

IBM research (via Latent Space AINews recap) showed that extracting reusable strategy, recovery, and optimization tips from agent trajectories improved AppWorld task completion from 69.6% to 73.2% and scenario goal completion from 50.0% to 64.3%, with the largest gains on hard tasks. A separate paper reframes multi-agent memory as a computer-architecture problem — cache hierarchy, coherence, access control — rather than a context-window problem.

> The computer-architecture framing is the useful one. "More context" is a hardware-constrained problem (as Latent Space argues, 1M tokens may be the ceiling for years). Memory as a structured system with coherence and access-control properties is where the real agent quality improvements will come from. For enterprise AI, this maps directly onto how you'd design a knowledge graph or episodic store for a long-running agent.

### [Nyne Gives AI Agents the Human Context They're Missing](https://techcrunch.com/2026/03/13/nyne-founded-by-a-father-son-duo-gives-ai-agents-the-human-context-theyre-missing/)

Data infrastructure startup Nyne raised $5.3M in seed funding (Wischoff Ventures, South Park Commons) to solve the missing-context problem in AI agents. The company builds infrastructure that gives agents access to user-specific, relationship-aware context — the kind of information that currently lives only in a person's head.

> The problem is real: agents can call APIs, browse the web, and write code, but they don't know who they're helping or why. Nyne's bet is that context infrastructure becomes a foundational layer, not a feature. In enterprise, this overlaps with identity, CRM, and long-term memory — the same convergence point several larger vendors are circling. Worth watching as a signal of where the agent infra stack is heading.
