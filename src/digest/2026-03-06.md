### [OpenAI Launches GPT-5.4 with Pro and Thinking Versions](https://techcrunch.com/2026/03/05/openai-launches-gpt-5-4-with-pro-and-thinking-versions/)

GPT-5.4 and GPT-5.4 Pro launched across ChatGPT, API, and Codex — the first model to unify mainline reasoning and frontier coding (previously separate via Codex). It features a 1M token context window, native Computer Use (75% on OSWorld-Verified, above human baseline of 72.4%), and an 83% win-rate vs. professionals on the GDPVal knowledge work benchmark. GPT-5.4 also improves significantly on spreadsheet, document, and presentation tasks — scoring 87.3% vs 68.4% for GPT-5.2 on an internal banking analyst benchmark.

> The "it just works" moment for enterprise knowledge work automation has arrived. Unifying coding and reasoning into one model simplifies architectural choices considerably — no more choosing between Codex for dev tasks and mainline for reasoning. The OSWorld CUA numbers crossing human baseline make desktop and enterprise application automation use cases suddenly very credible for real deployments.

### [Agentic Manual Testing: Coding Agents That Verify Their Own Work](https://simonwillison.net/guides/agentic-engineering-patterns/agentic-manual-testing/#atom-everything)

Simon Willison's latest Agentic Engineering Patterns entry details how to direct coding agents to manually verify their own output — using `python -c`, `curl`, Playwright, and tools like Rodney (Chrome DevTools Protocol) and Showboat (which captures commands and their outputs as audit-ready documents). The argument: passing unit tests doesn't mean the code works, and agents should do the same manual checks a developer would.

> This is table-stakes engineering discipline for enterprise agentic development pipelines. Beyond quality assurance, "show your work" artifacts via Showboat are exactly the evidence compliance and audit teams will require when agentic-generated code touches regulated systems. Building this into your agent workflow from day one is far cheaper than retrofitting it.

### [Clinejection: Prompt Injection Compromises Cline's Production Releases](https://simonwillison.net/2026/Mar/6/clinejection/#atom-everything)

Adnan Khan disclosed a sophisticated attack chain against Cline's GitHub repository: a crafted issue title injected a prompt into the AI-powered issue triage workflow (running `claude-code-action`), which triggered `npm install` of a malicious package, stuffed the GitHub Actions cache with 11GB of junk to evict the legitimate cache, then poisoned the shared cache key used by the nightly release workflow — ultimately exfiltrating NPM publishing secrets. A malicious `cline@2.3.0` was published before the responsible disclosure was acted on.

> Every enterprise running AI-powered CI/CD automation needs to read this. Issue titles, PR descriptions, and comments are now attack vectors — inputs no one traditionally treated as adversarial. The takeaways are concrete: scope tool permissions tightly (no bash for issue triage), isolate workflow cache keys by workflow, and treat any user-supplied text that reaches an agent as untrusted input. Least-privilege for agentic tools is the new network perimeter.

The models are excellent today; the question is what happens in six months.
