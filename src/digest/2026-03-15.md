### [What is Agentic Engineering? — Simon Willison's New Guide](https://simonwillison.net/guides/agentic-engineering-patterns/what-is-agentic-engineering/)

Simon Willison published the opening chapter of his *Agentic Engineering Patterns* guide, defining "agentic engineering" as software development assisted by coding agents — systems that write and execute code in a loop to achieve a goal. He argues that writing code has never been the primary challenge; figuring out *what* code to write is. The guide is a work in progress, updating as techniques evolve.

> This is the reference definition to send colleagues still debating what "agentic" means in practice. Willison's framing — agents as tools that run in a loop, humans as architects of intent — maps directly to what enterprise teams are actually building. Worth bookmarking and following as new chapters land.

### [Memory Is Becoming the Differentiator for AI Agents (Latent Space AINews)](https://www.latent.space/p/ainews-context-drought)

IBM research showed that agents extracting reusable strategy and recovery tips from past trajectories improved AppWorld task completion from 69.6% to 73.2%, and scenario goal success from 50.0% to 64.3% — with the biggest gains on hard tasks. Separately, Hermes Agent shipped as a self-hostable agent platform with persistent, user-specific memory and skills that improve over time.

> The framing of multi-agent memory as a "computer architecture problem" — cache hierarchies, coherence, access control — is the right mental model for enterprise agent design. The IBM numbers are modest individually but point toward a real multiplier as agents learn from past runs rather than starting cold every session.

### [Shopify CEO Uses Coding Agent to Achieve 53% Speedup in 20-Year-Old Library](https://github.com/Shopify/liquid/pull/2056)

Shopify CEO Tobias Lütke used Karpathy's *autoresearch* pattern — a coding agent running approximately 120 automated experiments over two days — to find micro-optimizations in the Liquid template engine, yielding 53% faster parse+render and 61% fewer allocations in a codebase that has had hundreds of contributors over 20 years.

> The pattern is directly transferable to enterprise performance work: give an agent a benchmarking script, a clear goal ("make it faster"), and let it brainstorm and test hypotheses in a loop. The prerequisite is a solid test suite — 974 unit tests made this possible. Without them, none of it would have been trustworthy.

### [AgentArmor: Open-Source 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

A new open-source framework targeting the fact that most production AI agents run with no meaningful security layer beyond implicit model trust. AgentArmor wraps any agentic architecture with 8 independent layers: prompt injection detection, encrypted vector storage, context isolation, action risk scoring (READ=1 → DELETE=7 → EXECUTE=8 → ADMIN=10), network egress control, PII redaction via Microsoft Presidio, inter-agent HMAC authentication, and JIT permissions with short-lived credentials. Tested against all 10 OWASP ASI risks from the December 2025 spec.

> The action risk scoring and inter-agent authentication layers are the ones to study first. As agents gain write access to real enterprise systems — databases, APIs, code repos — "we trust the LLM" will not survive a security audit. This framework gives a concrete layered model to reason about and implement against.
