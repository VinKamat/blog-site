### [Anthropic Hands Claude Code Auto Mode — AI on a Leash](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/)

Anthropic shipped "auto mode" for Claude Code, a new permissions model where a separate Claude Sonnet 4.6 classifier reviews each action before it runs — blocking scope escalation, untrusted infrastructure targets, and prompt injection from external content. It's an explicit alternative to `--dangerously-skip-permissions`, with a published allow/deny ruleset you can inspect via `claude auto-mode defaults`.

> The classifier approach is non-deterministic by design, and Simon Willison is right to be skeptical — it won't catch supply chain attacks through pinned deps, and "AI reviewing AI" is a soft guarantee at best. But for enterprise teams that need *some* guardrails without full sandbox isolation, this is a real step. The default ruleset is surprisingly thoughtful: it distinguishes between declared deps (OK) and agent-chosen installs (blocked), and soft-denies direct pushes to main. Worth reading the defaults JSON before rolling this out.

### [LiteLLM PyPI Supply Chain Attack — 47,000 Compromised Downloads in 46 Minutes](https://simonwillison.net/2026/Mar/24/malicious-litellm/)

LiteLLM versions 1.82.7 and 1.82.8 on PyPI were compromised with a credential stealer hidden in a `.pth` file — which Python executes at interpreter startup, *before* any `import` statement. The attack exfiltrated SSH keys, AWS/Azure/kube configs, git credentials, shell history, crypto wallets, and more. PyPI quarantined the package after 46 minutes, but ~47,000 downloads had already landed. The compromise originated from a Trivy CI scanner exploit that yielded the CEO's PyPI credentials.

> This one stings because LiteLLM is load-bearing infrastructure for a huge swath of Python AI projects. The `.pth` trick is particularly nasty — standard code review doesn't catch it, Docker isolation actually *did* help here, and the blast radius extends to anything with LiteLLM as a transitive dep (DSPy, Aider's older pin was safe). The real lesson for enterprise AI teams: pin your deps, run cooldowns on updates, and don't trust `pip install` in CI without version locking. In an agentic world where the whole filesystem is in-context, supply chain is attack surface.

A real 6x reduction would meaningfully change cost calculus for RAG pipelines with large retrieval context. Worth tracking whether this lands in Azure OpenAI Service. The broader pattern: the performance gains this quarter are coming from runtime and memory systems, not just bigger checkpoints.

### [Thoughts on Slowing Down — Cognitive Debt in Agentic Engineering](https://simonwillison.net/2026/Mar/25/thoughts-on-slowing-the-fuck-down/)

Mario Zechner (creator of the Pi agent framework) published a sharp critique of current agentic coding practices: agents remove the human bottleneck, so small errors compound at an unsustainable rate before you notice. He argues for manually writing anything that defines architecture and APIs, and setting hard limits on how much agent-generated code you review per day. Simon Willison largely agrees, framing it as "cognitive debt."

> This is the most honest thing written about agentic coding in months. The demo-to-production gap in agentic systems is real, and a lot of it comes from exactly this: removing yourself from the loop feels like speed, but you're actually just deferring the reckoning. For enterprise AI architecture, this maps directly to why orchestration guardrails, human-in-the-loop approval steps, and bounded agent scopes matter — it's not timidity, it's engineering discipline. The right mental model is: agents accelerate execution, humans own architecture.

### [Figma MCP Server Open Beta — Design Canvases as Agent-Native Surfaces](https://www.latent.space/p/ainews-apples-war-on-slop)

Figma launched its MCP server in open beta, making design canvases directly editable by AI agents via standard MCP tool calls. GitHub Copilot CLI and Cursor both shipped same-day integrations — Cursor extended it to generating components in Figma using a team's design system. It's the clearest mainstream example yet of tool-calling becoming product-native rather than chat-wrapper-native.

> For teams building AI-assisted frontend workflows, this is significant. Not because AI-generated UIs are production-ready, but because it closes the loop from "describe a component" to "it's in your design system" without a human intermediary in every step. The pattern generalizes: MCP servers on production surfaces (Figma, Slack, CRMs) are the agentic integration layer that actually matters for enterprise tooling, more than custom API wrappers.
