### [Microsoft's new MAI models](https://simonwillison.net/2026/Jun/2/microsofts-new-models/)
Microsoft announced MAI-Thinking-1, a reasoning model with 1T total parameters and 35B active, plus MAI-Code-1-Flash, a 137B code model with 5B active that is rolling into GitHub Copilot in VS Code. Simon Willison also flagged that Microsoft's technical paper describes large-scale web crawl training data despite early language around clean, licensed data.
> Microsoft is pushing deeper into owning the model layer instead of staying a wrapper around other labs. The part that matters for enterprises is where these models land in Copilot, Azure controls, and cost per serious workload.

### [New Microsoft tool lets devs spin up AI behavior tests using text descriptions](https://techcrunch.com/2026/06/02/new-microsoft-tool-lets-devs-spin-up-ai-behavior-tests-using-text-descriptions/)
Microsoft released ASSERT, short for Adaptive Spec-driven Scoring for Evaluation and Regression Testing, as an open source framework for turning plain-language policies into scored AI behavior tests. It can generate scenarios, run evaluations against a target system, and capture tool calls and intermediate actions so teams can inspect failures.
> This is the kind of plumbing enterprise teams need before agents move from demos into durable systems. If ASSERT works well, it can shorten the gap between policy writing and regression coverage.
</div>

### [Microsoft offers devs a better way to control AI agent behavior](https://techcrunch.com/2026/06/02/microsoft-offers-devs-a-better-way-to-control-ai-agent-behavior/)
Microsoft introduced Agent Control Specification, an open source standard for defining what agents may do, what they must not do, when humans must approve actions, and what evidence gets logged. The SDK checks these policies at multiple interception points across an agent workflow and ships with integrations for several agent frameworks and tool stacks.
> Prompt-level guardrails were never going to be enough for enterprise agents with tool access. A portable policy layer is a stronger direction because governance has to travel with the agent across runtimes.
</div>

### [OpenAI launches new Codex tools for white-collar work](https://techcrunch.com/2026/06/02/openai-launches-new-codex-tools-for-white-collar-work/)
OpenAI launched six Codex plug-ins for data analytics, creative production, sales, product design, equity investing, and investment banking, along with Sites for hosted outputs and Annotations for finer-grained context. TechCrunch reports Codex has more than 5 million weekly active users, with knowledge workers now making up about 20 percent of users and growing faster than the developer base.
> OpenAI is moving from a coding assistant toward a configurable work platform. That raises the integration stakes, because enterprise value will depend less on raw model quality and more on context, permissions, and workflow fit.

### [Anthropic scales Claude Mythos to critical infrastructure in 15+ countries](https://techcrunch.com/2026/06/02/anthropic-scales-claude-mythos-to-critical-infrastructure-in-15-countries/)
Anthropic expanded Project Glasswing to about 150 organizations across more than 15 countries, giving critical infrastructure operators broader access to Claude Mythos for vulnerability discovery. The new cohort includes sectors such as power, water, healthcare, communications, and hardware where software compromise could affect very large populations.
> This is a serious signal that frontier labs want a permanent role in defensive security operations. If these models keep proving useful on real codebases, security review starts to look more like continuous machine-assisted analysis than periodic human audit.

### [Google rolls out fake call detection to protect against AI deepfake impersonation scams](https://techcrunch.com/2026/06/02/google-rolls-out-fake-call-detection-to-protect-against-ai-deepfake-impersonation-scams/)
Google is rolling out fake call detection in Phone by Google for Android 12+ devices, starting with Pixel, to identify AI voice impersonation scams. The feature uses a silent device verification step over RCS so a phone can check whether a trusted contact's device is truly placing the call.
> Consumer anti-spoofing features have enterprise implications too, especially for executive fraud and help desk abuse. Voice trust is getting harder, so device-backed verification is going to matter a lot more.
</div>
