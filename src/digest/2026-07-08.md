### [[AINews] Lilian Weng summarizes 35 papers on Harness Engineering for RSI](https://www.latent.space/p/ainews-lilian-weng-summarizes-35)
Latent Space highlighted Lilian Weng's new post on harness engineering for self-improvement, arguing that recursive self-improvement is increasingly about the system around the model instead of direct weight edits. The recap points to design patterns and papers ranging from ACE to newer meta-harness work, and ties that shift to product changes around background agents and managed execution.
> This is where agent architecture is getting real. Stronger models still need explicit goals, context, and control loops, and that keeps the harness squarely in the critical path for enterprise systems.

### [Better Models: Worse Tools](https://simonwillison.net/2026/Jul/4/better-models-worse-tools/)
Simon Willison surfaced Armin Ronacher's report that newer Claude models sometimes send malformed edit-tool arguments in Pi by inventing extra fields, even when the requested code change itself is correct. The post suggests tool-specific tuning can improve one harness while hurting compatibility with third-party schemas.
> This is a clean reminder that model quality and tool reliability are separate concerns. If a team treats tool calling as solved, they will miss regressions that only show up inside real orchestration flows.
</div>

### [Using DSPy to evaluate and improve Datasette Agent's SQL system prompts](https://simonwillison.net/2026/Jul/2/dspy-datasette-agent-prompts/)
Willison also published a DSPy-based evaluation project for Datasette Agent's read-only SQL question-answering prompt, using live tool calls, real prompts, and a gold-standard dataset. One concrete finding was that thin schema context pushed the model into guessing column names and retry loops, which made prompt fixes measurable instead of subjective.
> This is the kind of work more AI teams need to operationalize. Prompting gets better when it is attached to evals, traces, and failure analysis instead of taste.
</div>

### [Open Source AI Gap Map](https://simonwillison.net/2026/Jul/3/open-source-ai-gap-map/)
Current AI launched an open source AI Gap Map that catalogs 421 products across software, models, datasets, and hardware, plus a much larger long tail of uncategorized artifacts. Willison's main takeaway was the underlying MIT-licensed dataset and repo, which make the map useful as infrastructure instead of just a visual index.
> Open ecosystems are easier to take seriously when the metadata is structured and reusable. This kind of inventory helps architecture teams compare options without starting from vendor decks and vibes.

### [Hot French startup ZML releases free product to speed inference across lots of AI chips](https://techcrunch.com/2026/07/08/hot-french-startup-zml-releases-free-product-to-speed-inference-across-lots-of-ai-chips/)

### [Claude Cowork expands to mobile and web](https://techcrunch.com/2026/07/07/the-coding-agent-wars-are-spilling-into-the-rest-of-the-office-claude-cowork/)
Anthropic expanded Claude Cowork to mobile and web, letting users start a task on desktop, monitor it from a phone, and return later for the finished output. TechCrunch frames it as the coding-agent race spilling into the rest of office work, with the product behaving more like a background task runner than a chat box.
> The UX shift matters as much as the model shift. Once agents move into long-running, cross-device workflows, the real architecture question becomes state management, approvals, and recovery across sessions.
