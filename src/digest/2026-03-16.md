### [Anthropic Ships Opus 4.6 1M Context GA — Free for Enterprise, 78.3% MRCR](https://www.latent.space/p/ainews-context-drought)
Anthropic made 1M context windows GA for Max/Team/Enterprise users, removed the extra API charge for long context, dropped the beta header requirement, and bumped media limits to 600 images/PDF pages per request. The model scores 78.3% on MRCR v2 at 1M tokens — currently the best long-context retrieval number reported.
> This matters for enterprise RAG architectures: fewer chunking hacks, less retrieval complexity, more room to stuff in policy docs, code bases, and long conversation history. The price drop makes it practical to use at scale. That said, Latent Space is betting context windows stay capped near 1M for years — hardware constraints are real, and "context rationing" may become a real planning constraint for architects.

### [What Is Agentic Engineering? Simon Willison's New Guide](https://simonwillison.net/guides/agentic-engineering-patterns/what-is-agentic-engineering/)
Simon Willison published the opening chapter of a living guide on agentic engineering — the practice of building software with coding agents. He defines an agent precisely: software that runs tools in a loop to achieve a goal. The guide distinguishes agentic engineering from "vibe coding" (unreviewed prototype-quality LLM output) and argues that code execution is the critical capability that makes agents productive.
> The TDD-for-agents framing is worth internalizing: tests are effectively free now, so shipping untested agent-generated code is a choice, not a constraint. The guide also captures something important about the new role of the engineer — less time writing, more time specifying, verifying, and steering. For enterprise teams ramping up on AI-assisted development, this is a solid conceptual foundation.

### [AI Slop Kills Jazzband: The Slopocalypse Hits Open Source](https://jazzband.co/news/2026/03/14/sunsetting-jazzband)
Jazzband, the open-membership Python collective that shared push access across hundreds of maintainers, is shutting down. The reason: AI-generated spam PRs and issues have made shared push access untenable. Only 1 in 10 AI-generated PRs meet project standards; curl shut down its bug bounty after confirmation rates dropped below 5%; GitHub added a kill switch to disable pull requests entirely for repositories overwhelmed by AI slop.
> This is an early signal of a real governance problem that enterprise teams will face. AI-generated code contributions at scale create a review burden that can exceed the maintainers' capacity. The fix isn't a technical one — it's process: stricter contribution gates, better triage tooling, and honest conversations about what "open" means when contribution cost approaches zero.

### [Agent Memory Is Becoming the Real Differentiator](https://www.latent.space/p/ainews-context-drought)
IBM research shows that extracting reusable strategy, recovery, and optimization tips from agent trajectories improves AppWorld task completion from 69.6% to 73.2%, with scenario goal completion jumping from 50.0% to 64.3% — biggest gains on hard tasks. In parallel, a new paper reframes multi-agent memory as a computer architecture problem: cache/memory hierarchy, coherence, and access-control, rather than "just more context."
> This framing — memory as architecture, not afterthought — is where enterprise agentic systems are heading. The IBM numbers show that agents can self-improve by learning from their own trajectories, which has direct implications for long-running business process automation. Agents that accumulate operational knowledge over time will outpace stateless ones, even if the base model is identical.

### [AgentArmor: Open-Source 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)
AgentArmor wraps agentic architectures with 8 security layers targeting distinct attack surfaces: prompt injection detection (L1), encrypted vector DB storage (L2), instruction-data separation (L3), action risk scoring (L4), execution rate limiting and human approval gates (L5), PII redaction via Microsoft Presidio (L6), inter-agent HMAC auth (L7), and JIT permissions with short-lived credentials (L8). Tested against the full OWASP ASI spec (December 2025). Available as a Python library, FastAPI proxy, or CLI.
> Most production agent builds I've seen treat security as an afterthought bolted on after the demo succeeds — AgentArmor makes the layered model explicit. The action risk scoring (READ=1 through ADMIN=10) and human approval gates are particularly relevant for enterprise deployments where agents touch financial data or customer records. Worth watching even if you don't adopt it wholesale — the threat taxonomy alone is useful for architecture reviews.

### [MCP Ergonomics Debate + Chrome v146 Web MCP Support](https://www.latent.space/p/ainews-context-drought)
Chrome v146 adds native web MCP support, demonstrated with a LangChain Deep Agent that continuously browses X and compiles daily summaries. The broader MCP debate on X this week: MCP is strong for deterministic, centrally-maintained APIs with rapidly changing ground truth; lightweight "skills" (local natural-language procedures) are easier to deploy but more failure-prone. Usage numbers are still growing despite the death-pronouncements.
> MCP is becoming infrastructure. The Chrome integration is significant — it means browser-based agents can now participate in the same tool ecosystem as server-side agents without extra plumbing. For enterprise architects thinking about agent-to-system integration patterns, the MCP vs. skills distinction is a real design choice: how much determinism do you need, and who owns the tool definition?

### [Nyne Raises $5.3M to Give AI Agents Persistent Human Context](https://techcrunch.com/2026/03/13/nyne-founded-by-a-father-son-duo-gives-ai-agents-the-human-context-theyre-missing/)
Nyne, a data infrastructure startup, raised $5.3M seed led by Wischoff Ventures and South Park Commons. The company builds tooling to give AI agents persistent human context — the preferences, history, and behavioral patterns that currently reset with every conversation.
> The gap between demo agents and production agents often comes down to exactly this: a demo can look impressive with a clean slate, but real business workflows require agents that remember who the user is, what they've decided before, and what constraints apply to them specifically. Nyne is betting that human-context infrastructure becomes a layer every serious agent deployment needs. Early, but the problem is real.
