### [GPT-5.4 Mini and Nano: 76,000 Photos for $52](https://simonwillison.net/2026/Mar/17/mini-and-nano/)

OpenAI shipped GPT-5.4 mini and nano yesterday. Mini is 2x faster than the previous mini, targets coding and subagents, hits 400k context, and approaches GPT-5.4 benchmarks at $0.75/M input tokens. Nano undercuts Google's Gemini Flash-Lite at $0.20/M input — Simon Willison used it to describe a photo for 0.069 cents, putting a 76,000-photo collection within $52.

> The nano pricing is a real shift. Enterprise workflows that were previously cost-constrained for high-volume inference — document tagging, classification, RAG pre-processing — now have a credible path at scale. Mini's positioning as the default for background coding and subagent fan-out is the more immediate signal for agentic platform design.

### [OpenAI Codex Subagents Now GA](https://simonwillison.net/2026/Mar/16/codex-subagents/)

OpenAI moved Codex subagents from preview to general availability. They ship with default "explorer," "worker," and "default" roles, plus custom TOML-defined agents in `~/.codex/agents/` that can have their own system prompts and model assignments. Simon Willison also published a full [Agentic Engineering Patterns guide chapter on subagents](https://simonwillison.net/guides/agentic-engineering-patterns/subagents/) covering Claude Code, Cursor, VS Code Copilot, Gemini CLI, and others.

> Subagents are maturing from a curiosity into a standard architectural primitive. The TOML-based custom agent definition is clean — it's essentially giving teams a composable, version-controlled way to declare specialist agents. Worth paying attention to how this maps onto enterprise use cases: a code-reviewer agent, a compliance-checker agent, a data-pipeline debugger. The harness is becoming the product.

### [Mistral Small 4: 119B Parameters, Apache 2.0, Agentic Coding Built In](https://simonwillison.net/2026/Mar/16/mistral-small-4/)

Mistral released Small 4, a 119B MoE model (6.5B active parameters) under Apache 2.0. It merges three previously separate model lines — Magistral for reasoning, Pixtral for multimodal, and Devstral for agentic coding — into one. Supports `reasoning_effort="none"` or `"high"`, 256k context, and multimodal input.

> A single open-weight model unifying reasoning, vision, and coding is meaningfully different from maintaining a zoo of specialized models. For enterprise teams building self-hosted agentic pipelines on Azure or on-prem, this consolidation matters — fewer models to manage, fewer deployment configurations. The 6.5B active parameter count keeps inference costs competitive with much smaller models.

Anthropic shipped Claude Cowork, a desktop app built by Felix Rieseberg (ex-Slack, Electron specialist). Multiple observers — Simon Willison, Ethan Mollick — are comparing it favorably to OpenClaw. > Worth watching for enterprise reasons: Anthropic now has a full-stack story from API → Claude Code → Claude Cowork (desktop GUI). The sandboxing approach via Electron is interesting — it's a pragmatic answer to the "give the agent access to my machine" problem that doesn't require a separate VM or container. How this plays out on managed corporate devices will be the real test.

### [AgentArmor: 8-Layer Security Framework for AI Agents](https://github.com/Agastya910/agentarmor)

An open-source framework that wraps agentic architectures with 8 independent security layers: prompt injection detection, AES-256 storage encryption, instruction-data separation (like parameterized SQL for LLM context), action risk scoring, execution rate limits and human approval gates, PII redaction via Microsoft Presidio, inter-agent HMAC mutual auth, and JIT identity with short-lived credentials. Tested against the OWASP ASI spec from December 2025.

> This is the kind of scaffolding that belongs in every enterprise agentic deployment and almost never shows up in the demo. The L4 action risk scoring — READ=1, DELETE=7, EXECUTE=8, ADMIN=10 — is a practical pattern for building guardrails without rewriting agent logic. L7 inter-agent auth matters especially as multi-agent topologies become real production systems rather than architecture diagrams.

### [Cursor Trained RL-Based Context Compaction, Cut Errors 50%](https://x.com/cursor_ai/status/2033967614309835069)

Cursor trained its Composer model to self-summarize long coding sessions through reinforcement learning rather than prompting. The result: 50% fewer compaction errors and significantly better performance on long-horizon tasks that span many files and tool calls.

> This is a quiet but important architectural result. Prompting an LLM to summarize its own context is fragile — the model doesn't have a strong signal about what's actually important. RL-based compaction gives the model that signal through task outcomes. For enterprise coding agents doing multi-session work (refactoring a large codebase, running a long debugging loop), this directly affects reliability at the point where most agents fall apart.

</div>

### [Mistral Forge: Train Enterprise AI From Scratch on Your Own Data](https://techcrunch.com/2026/03/17/mistral-forge-nvidia-gtc-build-your-own-ai-enterprise/)

Announced at NVIDIA GTC, Mistral Forge lets enterprises train custom models from scratch on proprietary data — not fine-tuning, not RAG, but full pretraining on your corpus. Mistral is positioning this as a differentiated enterprise offering against OpenAI and Anthropic, which both rely on API access to shared models.

> The pitch is data sovereignty at the model layer, not just at the retrieval layer. For regulated industries — financial services, healthcare, legal — where the model itself needs to be owned and audited, this is a different conversation than "here's a fine-tuned adapter." The question is whether the economics work: full pretraining is expensive, and most enterprises don't have the data volumes where it pays off over fine-tuning. But the strategic positioning is clear.

### [World Launches Human Verification for AI Shopping Agents](https://techcrunch.com/2026/03/17/world-launches-tool-to-verify-humans-behind-ai-shopping-agents/)

Sam Altman's World (formerly Worldcoin) is expanding its biometric identity verification into agentic commerce — verifying that a human authorized a purchase, even when an AI agent executed it. As AI agents autonomously browse, compare, and buy, the question of "who is accountable for this transaction?" becomes a real compliance and fraud concern.

> This is the identity layer for the agentic era, and it's a genuinely hard problem. An agent acting on behalf of a human needs some provable link back to that human's authorization. World is one bet on how to solve it (biometrics + cryptographic proof). Enterprise architects building agentic commerce flows — or any agentic system that takes consequential actions — will need an answer to this, whether it's World or something else.
